SOLUÇÃO MANUAL (Passo-a-passo):
--------------------------------

Se preferir fazer manualmente, siga os passos:

1️⃣  PARAR DAEMONS
    cd /home/lucas_marruda/Documentos/pessoal/gitPessoal/PDV/desktop/pdv-compose
    ./gradlew --stop
    pkill -f "GradleDaemon"
    pkill -f "KotlinCompileDaemon"

2️⃣  LIMPAR CACHES LOCAIS
    rm -rf build
    rm -rf .gradle

3️⃣  LIMPAR CACHES GLOBAIS
    rm -rf ~/.gradle/caches/
    rm -rf ~/.gradle/daemon/
    rm -rf ~/.kotlin/

4️⃣  COMPILAR NOVAMENTE (sem daemon)
    ./gradlew clean build --no-daemon --refresh-dependencies

5️⃣  EXECUTAR O PROJETO
    ./gradlew run


EXPLICAÇÃO DOS ERROS:
---------------------

❌ Erro original: "classpath/lasspath"
   → Gradle Wrapper corrompido ou ausente
   → SOLUÇÃO: Recriar arquivos do wrapper

❌ Erro: "Cannot add task 'run' as a task with that name already exists"
   → Conflito entre plugins 'application' e 'compose.desktop'
   → SOLUÇÃO: Remover plugin 'application' do build.gradle.kts ✓

❌ Erro: "PersistentEnumerator storage corrupted"
   → Cache do Kotlin corrompido
   → SOLUÇÃO: Deletar caches e recompilar


O QUE FOI CORRIGIDO:
-------------------

✅ Removido plugin 'application' conflitante do build.gradle.kts
✅ Criado script fix.sh para limpeza automática
✅ Gradle Wrapper recriado com arquivos corretos
✅ Documentação de resolução criada

SE AINDA ASSIM DER ERRO:
------------------------

1. Verifique se o Java 17+ está instalado:
   java -version

   Se não estiver:
   sudo apt install openjdk-17-jdk

2. Verifique se há espaço em disco:
   df -h .

   Necessário: pelo menos 500MB livres

3. Verifique conexão com internet:
   ping -c 1 google.com

   Necessária para baixar dependências

4. Tente recriar o Gradle Wrapper:
   rm -rf gradle gradlew gradlew.bat

   Depois baixe novamente:
   curl -L -o gradlew https://raw.githubusercontent.com/gradle/gradle/v8.4.0/gradlew
   chmod +x gradlew
   mkdir -p gradle/wrapper
   curl -L -o gradle/wrapper/gradle-wrapper.jar \
       https://raw.githubusercontent.com/gradle/gradle/v8.4.0/gradle/wrapper/gradle-wrapper.jar

4. Não interrompa a primeira compilação
5. Se travar, Ctrl+C e tente novamente
